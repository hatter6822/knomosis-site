About

Knomosis brings together three codebases that are usually read with three different toolchains. This site renders each one as the same kind of adaptive flow chart, so the structure of the Lean proofs, the Rust runtime, and the Solidity contracts can all be navigated the same way.

One graph, every language

Each map is generated from a JSON codemap of the repository. Modules become nodes; cross-module references become edges. The viewer derives the dependency and call graph directly from the data — nothing is hand-drawn.

Module and declaration views

Start at a module to see what it depends on and what depends on it. Drill into any declaration to see exactly what it calls and what calls it, with edges marking cross-module references.

Dependency-minimized

The whole site is vanilla HTML, CSS, and JavaScript — no frameworks, no build step. It mirrors the architecture of the seLe4n project site it was adapted from.

How the viewer works

Choose a codebase from the toolbar, then search or click to select a module. The selected module sits in the center lane; the modules it depends on appear on the left and the modules that depend on it appear on the right. The interior menu lists every declaration in the module, grouped by kind. Click a declaration to switch into its call graph, and use the breadcrumb to return.

Counts above reflect the bundled snapshot; the viewer always shows live statistics for the selected codebase.
